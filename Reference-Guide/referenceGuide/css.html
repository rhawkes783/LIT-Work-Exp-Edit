CSS - Cascading Style Sheets

Making the web look pretty since 1996

CSS is the language for describing and altering presentation of web pages(colours, fonts, data layout, etc). It gives designers and developers the ability to adapt the presentation of a web page to different types of devices, such as large screens, small screens, or printers. CSS is independent of HTML and can be used with any XML-based markup language. The separation of HTML from CSS makes it easier to maintain sites(especially when something stops working), tailor pages to different environments/devices, etc. This is known as the separation of structure (or content) from presentation.

CSS Selectors

Before we get into explaining how CSS can be applied to a web site, we must first explain how CSS works. At a glance, it's very simple. CSS styling is applied to elements on a web page using CSS selectors. Below is a simple example of a CSS selector that will change the background colour of all <div> elements to green. 

div { background-color: green; }

Selectors are the backbone of CSS and how it works. Once you know how to apply these selectors then you can use CSS to enhance the presentation of a web page or web site. Below is an example of the <div> before styling and then after styling has been applied.

How to apply CSS

There are 3 main ways to apply CSS to your web pages. They are in-line, internal and external. Firstly, it's important to note that whilst CSS isn't necessarily governed on the web, it is important to make sure we use it properly on web pages. The separation of CSS files from HTML markup helps ensure HTML is readible and accessible.

Accessible HTML means that disabled users(eg. a blind person) can still access the web and engage with the content through devices known as 'screen readers'. These devices scan all the elements on the page and read the content back to the user. If we are to add styles to our HTML directly, it can affect the result that is read back to the user. Moving aside from accessibility, mixing CSS with HTML can be very difficult to debug for web designers and developers if a problem arises on a web page.

As mentioned above, CSS isn't heavily-governed on the web and this is why various browsers can display the same web page in different ways. The more you work with CSS(especially in older web browsers), the more familiar you will become with how Google's Chrome browser will display things in a different manner to Microsoft's Internet Explorer browser. However, this is a topic that you will learn as you get more familiar with CSS.

There are 3 ways that you can include CSS on a web page. We'd recommend that you go with option 3 - External style sheets, but it's important that we explain each way to apply them.

In-line

In-line styles are added directly into the HTML markup. Below is a simple example to highlight how an in-line works. 

                    
<h1 style="color: green">Guardians of the Galaxy</h1>
This will make the header text green.

Internal

Internal styles are added in the head element in the HTML of a web page. They are wrapped in <style> tags. 

                    
<html>
    <head>
        <title>Guardians of the Galaxy</title>
        
        <style>

            h1 { color: green; }
            h2 { color: blue; }
        
        </style>
        ...

This will make the header text for h1 header text green and header text for h2 header text blue.

External

External styles are contained in a separate file and can be used across an entire web site, not just for one single page. This makes external styling a very popular option and something that most web developers do. Below is an example of a basic style that will be saved in the file 'style.css'. 

h1 { color: green; }

style.css must be referenced from the HTML page you wish to apply the styling to. 

                        
<html>
    <head>
        <title></title>
        <link rel="stylesheet" href="style.css">
        ...
