GitHub

Web-based source control

GitHub is an open source project started by Linux creator Linus Torvalds. Source control lets you manage and store revisions of projects. This allows for developers to access previous versions of their code and 'rollback' changes if they feel recent developments have caused issues in a project. It's handy to know that you always have a backup of your project on GitHub as it means you can rely on it in case difficulties arise in a project. The service is primarily used to manage code but can be used to manage any other type of file(eg. PowerPoint presentations, Word documents, etc).

The main functionality of GitHub is 'forking'. This is when one user copies a repository from their account to another. You can then share your changes with the owner of the orignal repository through a notification called a 'pull request' to the original owner. The user can then simply merge the changes found in your repository with the original repository. One of the biggest advantages in using GitHub is it's simplicity.

During the development of this project, developers worked on the application using GitHub, working off a main repository where users 'forked' it and then made pull requests to committ their changes. GitHub has a great social aspect to its service and helps developers all over the world engage with one another whilst helping improve code.
