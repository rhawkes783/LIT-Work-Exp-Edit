Node

Sever-side JavaScript

Node is a relatively new scripting development language which is booming into popularity due to its ability to be written and learnt quickly. As you will see within the project, the use of JS within web development is pretty fundamental and has many uses (basic JS, JQuery, AJAX, JSON etc.). Node took the existing language of JavaScript and allowed it to be run on the Chrome V8 engine on the server. Due to the speed at which it can be created, we are going to use it for our project. For examples on how to write JS for node check out the JS cheat sheet. It is worthwhile noting that JQuery is purely client side and has no place within node.
On a server, such as the API which we are creating, there are a few key concepts that you need to know:

  • Http Requests and Responses
  • Routing
  • Asynchronous Execution

HTTP Request and Response

HTTP (Hyper-text transfer protocol) is the basis for all information passed around on the internet. When you request a webpage, your browser generates an http request and sends it to the required URL. It then waits until a response is recieved and displays the data returned (i.e. the html/css/js). We are creating our own http request and responses, the AJAX creates the request which is sent to our node server, which then generates the response which is passed back to the client.
In this way our data is being sent to and from the client and the server. Now the important thing to note here is that there are different types of request that can be made, depending on what the client needs. There are:

  • GET requests which are used to get specific information from the server
  • POST requests which take in data, which is encrypted and sent to the server to be saved
  • DELETE requests which tell the server to delete existing data records
  • PUT requests are used to replace the current data with the new data provided
There are other types available, but they are less commonly used. Within our server we need to specify the type of request for each url. An example of this (POST) can be seen in the next section.

Routing

A node server is a very simple concept. It will recieve Http Requests and will send out an Http Response. When a request is recieved, it includes a specific URL which provides the route for the server to follow in order to provide the client with the information that it requires. The server is setup in such a way that it can take the URL and can decipher which block of code or function to call for that route. We are using a library called "express" to handle our routing for us. We tell the express router that we want to create a specific type of route for the type of request that the server will recieve (GET, POST etc.). When we set up the route, we provide the URL that the client must call to access this route and a function which holds the code to run when the route is called. This function takes in two parameters, the http request and response objects. 


router.post('url/of/the/route', function(requestObject, responseObject){// Code to run; });

Asynchronous Execution

This is the key to why node is succesful as a server. Unlike the client JS, where execution is sequential, the node server allows us to deal with multiple server calls at the same time. This is vital, as a server must be able to deal with multiple different clients hitting it at the same moment and be able to deal with their request quickly and effectively. Our server.js file defines the server and it's execution cycle. It sets it up and once created, it responds to several different requests simultaneously.
