APIs

Software-to-software interfaces

An Application-Programming Interface(API) is a set of programming instructions used for accessing data from a web site/application. A good example of this would be a web site that offers knowledge and reviews on vintage Playstation One games; the web site might display eBay listings of the games down the side of the page so when a user is reading about a game they can also see how much it would cost if they wanted to purchase a copy from eBay. To do this, the web site must interact with the eBay API - Eg. Every time a user visits a web page for a specific game, the page would send the game title to the eBay API in a request and the API would return information to the page based on the game information sent.

eBay is just one example of how many large companies(Amazon, BBC, DropBox, etc) are releasing APIs that will let third-party developers interact with their services. APIs are becoming more and more popular on the world wide web as they increase the amount of data available to the user. In the case of our car insurance application, we are requesting data from the API when we click the 'retrieve quote' button. Our car insurance application makes a request to the API and in doing so, sends some information(eg. Name, gender, age, car model, etc) with the request. The API then receives the data, calculates a quotation value based on the information received and responds to the application with that value.

APIs can be written in many different programming languages. In our case, we developed the car insurance API using JavaScript and Node. In the code snippet below, we are making an AJAX call to the API, passing information gathered from the page(user & car details). The information being sent is in the data key(gender, age, noClaimsBonus, etc). Once the value has been returned from the API, it is then put into the HTML element 'txtQuote' and displayed on the page for the user to view. 


$.ajax({
    type: "GET",
    url: "http://example.com/api/calculateRates",
    data: {gender:gender, age:age, noClaimsBonus:yearsNoClaims, costOfCar:costOfCar, insuranceDuration:'12', carStorage:carStorage}
  }).done(function(msg) {
    $("#txtQuote").text(msg.result);
    showQuoteDetails();
});

Whilst this might look rather daunting, once you understand the core concepts of how an API works, it will all make sense. This section has been created just to give you a very brief overview into how APIs work and how they can be very beneficial to developers when they are wanting to provide users with data from various services.
